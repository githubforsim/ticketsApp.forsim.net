11/************************************************
12 * Contrôleur TicketController gérant les actions liées aux tickets dans l'application.
13 *
14 * Ce contrôleur gère toutes les actions liées aux tickets, y compris la création,
15 * la modification, la suppression, et l'affichage des tickets. Il interagit avec le modèle
16 * TicketModel pour exécuter ces actions et avec la vue pour afficher les résultats.
17 *
18 * Ce contrôleur inclut les fonctionnalités suivantes :
19 * - Affichage de la liste des tickets ouverts, fermés, résolus, etc., pour un produit spécifique.
20 * - Affichage des détails d'un ticket spécifique (titre, description, statut, etc.).
21 * - Ajout de nouveaux tickets dans la base de données.
22 * - Modification des détails d'un ticket existant (titre, description).
23 * - Gestion des pièces jointes associées à un ticket.
24 * - Envoi et récupération de messages dans le contexte d'un ticket spécifique.
25 * - Enregistrement des événements et historique des modifications d'un ticket.
26 *
27 * La classe utilise le modèle TicketModel pour effectuer les opérations CRUD (Create, Read, Update, Delete)
28 * sur les tickets. Les méthodes de ce contrôleur orchestrent la logique métier et l'intégration des données
29 * avec la vue et le modèle, en assurant que les actions sur les tickets sont sécurisées et cohérentes.
30 ***********************************************/

82 /************************************************
83 * Fonction qui permet d'obtenir le nom filtré de la pièce jointe et vérifie l'extension du fichier.
84 *
85 * @param array $attachment Les informations sur la pièce jointe téléchargée
86 * @return string Le nom de fichier nettoyé et vérifié
87 * @throws Exception Si une erreur survient lors du téléchargement ou si l'extension du fichier est invalide
88 ***********************************************/
89 private function processAttachment($attachment)
90 {
91 //On vérifie s'il y a eu une erreur lors de l'upload du fichier
92 if ($attachment['error'] !== UPLOAD_ERR_OK) {
93 throw new Exception('An error occurred with the file upload.');
94 }
95 //On récupère le nom original du fichier
96 $original_file_name = pathinfo($attachment['name'], PATHINFO_FILENAME);
97
98 //On récupère l'extension du fichier'
99 $extension = pathinfo($attachment['name'], PATHINFO_EXTENSION);
100
101 // Tableau contenant les extensions autorisées
102 $allowedExtensions = ['jpg', 'jpeg', 'png', 'pdf', 'doc', 'docx', 'zip', 'rar'];
103
104 //On vérifie si l'extension du fichier est une extension autorisé
105 if (!in_array($extension, $allowedExtensions)) {
106 throw new Exception('Invalid file extension. Only JPG, PNG, DOCX, ZIP, RAR, and PDF files are allowed.');
107 }
108
109 //Générer le nom de la pièce jointe épurée en combinant le nom du fichier d'origine et l'extension
110 $attachment_name = $original_file_name . '.' . $extension;
111 //On renvoi le nom de fichier nettoyé
112 return $attachment_name;
113 }
